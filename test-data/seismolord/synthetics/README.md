# Seismolord synthetics goldens

Committed golden files for the LAS-driven synthetics engine
(`src/pages/apps/Seismolord/engine/synthetics.js`), generated by the
Python oracle `tools/validation/seismolord/synthetics/gen_synthetics.py`
(numpy; run from `tools/validation/seismolord` with its `.venv`).
The oracle self-asserts its own math (hand-checked wedge reflection
coefficients, analytic Ricker peak/zero crossings, recovered bulk shift,
zero-phase symmetry of the extracted wavelet) before writing anything.

Consumed by `src/pages/apps/Seismolord/__tests__/synthetics.test.js`
(tolerance ~1e-5 relative; the JS engine is float32 end-to-end while the
oracle is float64).

Conventions shared by both sides (documented in full in the oracle):

- gaps: not finite (LAS parser NULL -> NaN), |v| >= 9.0e29 (seismic null
  1.0E+30), or a raw LAS null (-999.25, -999, -9999, -9999.25). Gaps are
  JSON `null` here and NaN in the JS engine.
- convolution: 'same', zero-fill gaps, validity mask = input validity.
- checkshot T(z): piecewise linear with end extrapolation — the exact
  `makeTvdssToTwt` checkshots branch of `engine/wellSection.js`.
- MD -> TVDSS: minimum curvature (`tools/validation/seismolord/wells/
  mincurve.py`, itself validated on a published worked example).

| file | case |
| --- | --- |
| `ricker.json` | (a) analytic Ricker 25 Hz, dt 2 ms, half-length 60 ms |
| `wedge.json` | (b) 3-layer wedge: hand-checked RCs placed in TWT via a known checkshot function, convolved |
| `las_pipeline.json` | (c) full pipeline: DT (US/M, with -999.25 AND 1.0E+30 gap zones) + RHOB down a deviated path, checkshots, resample, RC, convolve, validity |
| `wavelet_extract.json` | (d) statistical wavelet extraction reference (autocorrelation -> tapered/smoothed amplitude spectrum -> zero-phase wavelet) |
| `bulk_shift.json` | (e) normalized cross-correlation bulk shift: seismic = synthetic delayed 8 ms + noise + a null gap |
