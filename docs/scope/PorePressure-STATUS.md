## What shipped

  reference + `test-data/porepressure/goldens.json` (anchors A1–A8
  self-asserted at generation, byte-identical regeneration). The
  synthetic-well golden is forward-inverse-consistent: the dt log is
  generated by inverting Eaton on an imposed PP profile.
- **P1 engine** — `src/pages/apps/PorePressureStudio/engine/`:
  pressures (hydrostatic + trapezoid overburden), gardner, nct (exact
  log-transform fit), eaton, bowers (loading/unloading, published
  ft/s-psi domain, SI edges), fracgrad (coefficient form: Eaton ν and
  Matthews-Kelly Ko are one identity), profile (pipeline with
  per-sample density provenance + Gardner fallback), velocitySource
  (P4). Unphysical inputs throw. Jest vs goldens at 1e-12 (well
  recovery 1e-9).
- **P2 persistence** — `pp_projects` (migration `20260714170000`,
  applied live, rp_projects owner-only pattern; RLS pentest 5/5,
  `tools/validation/porepressure/rls-pentest.sql`). Pentest driver
  gotcha recorded there: management-API returns only the last
  statement's rows — probes run as separate calls ending on the
  select.
- **P3 workstation** — PPWorkstation on the shared WorkspaceShell,
  injected backends (inMemoryBackend imports goldens.json directly;
  registryBackend = wellsRegistry + pp_projects). White-chartTheme
  prognosis chart (OBG/Ph/PP/FP + calibration dots), NCT view with
  deterministic picks + fit-back, params dock, ribbon depth readout
  in MPa. `/dev/pore-pressure-studio` harness.
- **P4 integration + close-out** —
  - Seismolord velocity trends: per-volume `manifest.velocity`
    single-function (v0 + k·z) models sampled into pseudo-sonic
