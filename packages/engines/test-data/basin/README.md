# BasinFlow G7 validation goldens

Generated by `tools/validation/basin/genfixtures.py` (stdlib Python,
deterministic — regeneration must be byte-identical). The generator
self-asserts 12 analytic/behavioural anchors (A1–A12, see its docstring)
before writing; a regeneration that breaks physics refuses to land.

`goldens.json` contents:
- `decompaction` — Athy solid-thickness / decompacted-thickness cases.
- `heat_two_layer_steady` — exact piecewise-linear steady profile.
- `easyro_ramps` — Easy%Ro (Sweeney & Burnham 1990) under 1/3/10 °C/Ma
  linear heating, tabulated per °C.
- `kerogen_isothermal_tr` — parallel-reaction TR closed-form cases.
- `reference_basin` — full forward run (4 layers, erosion event,
  declining paleo-heat-flow), series decimated to every 5 Ma, plus
  no-erosion and constant-Q control end-points.

The model spec (constants, grid, boundary conditions, erosion phantom
semantics, expulsion bucket) is pinned in
`tools/validation/basin/oracle.py` — the G7.1 JS engine implements
that spec and its jest suite compares against these goldens. The oracle
must never import from or be checked against the JS (dual-implementation
rule).
